Ноды:

    1. Отличаются по типу:
        - фрц       1
        - рц        2
        - пвз       3
        - франшиза  4
        - поставщик 5

    2. Отличаются по статусу:
        - активный 
        - неактивный

    пример:

    {
        id: "00000000-0000-0000-0000-000000000000",
        name: "название офиса",
        type: 2,
        is_active: 1
    }

Ребра:

    5 состояний:
        - только ребро
        - ребро + активное расписание
        - ребро + неактивное расписание
        - активное расписание
        - неактивное расписание

    кодировка состояний:
    - первая цифра отвечает за ребро:
        1 - есть,
        2 - нет
    - вторая цифра отвечает за расписание:
        0 - нет расписания,
        1 - есть активное расписание,
        2 - есть неактивное расписание

    Таким образом, полчаются сочетания:
    10
    11
    12
    01
    02

    отображаются следующим образом:

    ребро: есть - сплошная линия, нет - пунктирная
    расписание: нет - серый, есть активное - зеленый, есть неактивное - оранжевый

    10 - есть ребро, нет расписания - сплошная линия серого цвета
    11 - есть ребро, есть активное расписание - сплошная линия зеленого цвета
    12 - есть ребро, есть неактивное расписание - сплошная линия оранжевого цвета
    01 - нет ребра, есть активное расписание - пунктирная линия зеленого цвета
    02 - нет ребра, есть неактивное расписание - пунктирная линия оранжевого цвета

    пример:

    {
        "source": "00000000-0000-0000-0000-000000000000",
        "target": "00000000-0000-0000-0000-000000000001",
        "status": "11"
    }
